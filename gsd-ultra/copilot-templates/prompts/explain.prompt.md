Explain the selected code in detail at a ${input:level:beginner/intermediate/advanced} level.

## Explanation Should Include

1. **Overview**
   - What does this code do? (high-level purpose)
   - When/why would you use it?

2. **Step-by-Step Breakdown**
   - Walk through the code line by line
   - Explain each significant operation
   - Describe the flow of data

3. **Key Concepts**
   - Programming patterns used
   - Language-specific features
   - Important design decisions

4. **Dependencies & Context**
   - What does this code depend on?
   - How does it fit into the larger system?
   - What calls this code?

5. **Potential Gotchas**
   - Common mistakes when using this code
   - Edge cases to be aware of
   - Performance considerations

## Format

Use clear headings and bullet points.
Include simplified code examples where helpful.
For complex logic, use diagrams or step-by-step examples with sample data.
